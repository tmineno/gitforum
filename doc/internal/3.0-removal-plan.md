# v3.0.0 Per-File Removal Plan

Source of truth for Phase 4 removal. Bound by ADR-011.

Classification:

- `KEEP` — used by 3.0 runtime as-is (may be edited; file path stays)
- `KEEP-RENAME` — surface stays, contents heavily rewritten or path changes
- `MOVE-TO-LEGACY` — survives only inside `src/internal/legacy/` and is
  reachable only from the migrate command path
- `DELETE` — removed at Phase 4

Phase reference:

- Phase 0 = preparatory (tasks `lenm78ma`, `xl8jnot9`, `4w8hm98j`,
  `hib03vrq`, `3dx6szoh`)
- Phase 1 = additive snapshot path
- Phase 2 = command cutover
- Phase 3 = migration command
- Phase 4 = legacy deletion

## Legacy logic inside KEEP files

are clean. The current `v3.0.0-draft` tip has substantial v1.x/v2.x logic
embedded inside files this plan classifies as KEEP. Every reference must
be removed during Phase 1 or Phase 2; the build-time gate (task
`3dx6szoh`) is what guarantees no residue survives Phase 4.

#### Phase 4 sweep

After Phase 2 ends, only `migrate.rs` should reach into
`internal::legacy::*`. The build-time gate enforces this. Phase 4 then:
1. Runs `git rm -r src/internal/index.rs src/internal/reindex.rs`
2. Runs `git rm src/internal/github*.rs`
3. Runs `git rm src/internal/state_change.rs src/internal/write_ops.rs src/internal/create.rs src/internal/repair.rs src/internal/repair_workflow.rs src/internal/prune.rs src/internal/purge.rs src/internal/timeline.rs src/internal/commands/repair_workflow.rs`
4. Re-runs the contamination audit; expects zero hits in non-`legacy/` paths
   for every pattern in the table above.

### Heuristic for "is this KEEP file done?"

A KEEP file is considered Phase-2-clean when:
1. It does not contain `use super::event`, `use super::workflow`, `use super::index`, `use super::write_ops`, `use super::timeline`, `use super::create`, `use super::state_change`, `use super::repair*`, `use super::prune`, `use super::purge`, `use super::github*`.
2. It does not reference `EventType`, `DomainEvent`, `Lifecycle`, `KIND_PRESETS`, `facet*`, `replay`, `materialize` outside string literals or doc comments.
3. The build-time gate from task `3dx6szoh` accepts the file.

`migrate.rs` is exempt — it is the migration entry point and is allowed
to import `internal::legacy::*`.
