# v3.0.0 Per-File Removal Plan

Source of truth for Phase 4 removal. Bound by ADR-011.

Classification:

- `KEEP` â€” used by 3.0 runtime as-is (may be edited; file path stays)
- `KEEP-RENAME` â€” surface stays, contents heavily rewritten or path changes
- `MOVE-TO-LEGACY` â€” survives only inside `src/internal/legacy/` and is
  reachable only from the migrate command path
- `DELETE` â€” removed at Phase 4

Phase reference:

- Phase 0 = preparatory (tasks `lenm78ma`, `xl8jnot9`, `0exewp7g`,
  `t8o3vnt6`, `4w8hm98j`, `hib03vrq`, `3dx6szoh`)
- Phase 1 = additive snapshot path
- Phase 2 = command cutover
- Phase 3 = migration command
- Phase 4 = DELETE-list execution; `internal::legacy/` is **retained**
  for the migrate command per ADR-011 Decision 1

### Heuristic for "is this KEEP file done?"

A KEEP file is considered Phase-2-clean when:
1. It does not contain `use super::event`, `use super::workflow`, `use super::index`, `use super::write_ops`, `use super::timeline`, `use super::create`, `use super::state_change`, `use super::repair*`, `use super::prune`, `use super::purge`, `use super::github*`.

3. The build-time gate from task `3dx6szoh` accepts the file.

`src/main.rs` and `src/internal/commands/migrate.rs` use the same
heuristic with two adjustments:
- main.rs uses `use git_forum::internal::*` form rather than `use super::*`;
